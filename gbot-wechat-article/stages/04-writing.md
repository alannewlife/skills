# 阶段04：全文撰写

## 目标
根据框架完成2500-3000字文章初稿

---

## 写作风格（必须遵守）

### 1. 口语化表达
```
❌ 书面： "我们需要深入思考这个问题，才能得出有效结论。"
✅ 口语： "这个问题得好好想想，不然容易踩坑。"
```

### 2. 情绪外放
```
✅ 太爽了！终于搞定了这个问题！
✅ 真的气死我了，这个坑我踩了三遍！
✅ 超级兴奋！刚发现一个神仙工具！
```

### 3. 完整句子表达（重要！）
```
❌ 过于碎片化：
昨天研究这个问题。

发现一个奇怪现象。

参数都调好了。

结果反而更糟。

✅ 完整表达：
昨天研究这个问题的时候，我发现了一个奇怪的现象：参数都调好了，结果反而更糟。我困惑了好久，反复检查了几遍流程，最后才发现——原来一开始就搞错了。
```

### 4. 实用主义
```
少废话，多干货
删掉没有信息量的段落
```

### 5. 真实体验
```
敢吐槽，敢承认不足
分享踩坑经历
用第一人称"我"
```

---

## 执行步骤

### 步骤1：撰写开篇（150-250字）

**开篇目标：3秒抓住读者**

选择一种方式：

**方式A：痛点共鸣**
```
"昨天，一个粉丝私信我：

'老师，我已经坚持XXX一个月了，为什么还是XXX？'

这个问题，让我想了很久..."
```

**方式B：数据震撼**
```
"90%的XXX者，都在第3个月放弃。

这是我在行业里观察了5年，得出的数据。

说实话，看到这个数字的时候，我也挺惊讶的。"
```

**方式C：场景代入**
```
"周五晚上10点，我刚关掉电脑，手机弹出一条消息：XXX

我当时就坐不住了，立马打开电脑继续干。"
```

**方式D：金句开场**
```
"'XXX，是最好的XXX。'

这句话我听了100遍，直到昨天，我才真正理解了它的意思。"
```

---

### 步骤2：撰写正文（2000-2500字）

**写作原则：**

1. **用完整句子表达** - 避免短句碎片化
2. **用具体例子** - 不要只讲抽象概念
3. **关键信息用行内引用** - 用反引号标识重点，帮助快速扫描
4. **适当留白** - 段落间空行
5. **加入情绪** - 爽/不爽/兴奋
6. **真实体验** - 我的踩坑经历

**示例对比：**
```
❌ 碎片化： "我们需要坚持学习。

不断提升自己。

才能适应社会。"

✅ 完整表达： "我们需要坚持学习，不断提升自己，才能适应这个快速变化的社会。"

❌ 抽象： "这个方法很有效。"
✅ 情绪+具体： "这招真管用！我亲测了一周，效果绝了！"
```

---

### 步骤3：撰写结尾（200-350字）

**结尾目标：留下印象或引导行动**

选择一种方式：

**方式A：总结+号召**
```
"回顾一下，今天分享了：

1. XXX
2. XXX
3. XXX

如果觉得有用，点个「在看」，让更多人看到~

你在XXX中遇到过什么问题？

欢迎在评论区分享！"
```

**方式B：金句升华**
```
"XXX的路上，没有捷径。

但每一步，都算数。

与君共勉。"
```

**方式C：互动提问**
```
"你在XXX中遇到过什么问题？

欢迎在评论区分享~

我会选出3位朋友，送出XXX"
```

---

### 步骤4：自我检查

完成初稿后，检查以下问题：

```
□ 开篇是否有吸引力？
□ 用完整句子表达了吗？（避免短句碎片化）
□ 段落之间有逻辑连接吗？
□ 是否有具体例子？
□ 是否有情绪表达？（爽/不爽/兴奋）
□ 是否有真实体验？（我的经历/踩坑）
□ 逻辑是否清晰？
□ 是否有错别字？
□ 结尾是否有行动号召？
□ 字数是否达到2500-3000？
□ 关键重点是否使用行内引用反引号，而不是加粗？
```

### 步骤4.5：轻量层次增强

完成正文后，必须稍微修饰一下文字层次：

- 用反引号标出少量关键判断，例如：`担心没有用，判断才有用。`
- 可以加少量引用块承托金句，但不要密集。
- 不要用 `**重点**` 作为重点标识。
- 不改文章结构，不大幅重写正文。

---

### 步骤5：保存文章（自动执行）

文章完成后，自动执行：

```
【保存文章】

文件名：YYYYMMDD_文章标题关键词.md
路径：当前工作目录的 `outputs/`，或用户明确指定的目录
格式：排版后的Markdown
```

**排版要求：**
- 使用标准Markdown语法
- 标题层级清晰（# ## ###）
- 引用块、表格、列表格式正确
- 适当留白，便于阅读

---

### 步骤6：确认配图处理方式并生成提示词

文章保存前，执行以下流程：

**6.1 先确认是否直接出图**

如果用户已经明确要求“生成图片 / 开始画图 / 用 Agnes 出图 / 直接画出来”，不再追问；“直接画出来”默认就是用 Agnes Image 2.1 Flash 出图。

如果用户已经明确要求“只要提示词 / 先不要生成图片 / 不要出图”，不再追问，只生成文章和配图提示词。

如果用户没有明确说明，在生成最终 Markdown 前先用中文询问：

```text
这次配图要直接画出来，还是只提供图片提示词？

你可以回复：
1. 直接画出来：我会默认用 Agnes Image 2.1 Flash 生成图片并插入 Markdown。
2. 只提供提示词：我只写文章和配图 Prompt，不生成图片。
```

等用户选择后再继续。面向用户的确认问题、状态说明和交付说明都用中文；图片生成 Prompt 仍以英文为主，只描述画面。

**6.2 自动生成配图提示词并插入文章**
```
根据文章内容，自动生成：
1. 封面图提示词
2. 2-3张正文信息图提示词
3. 结尾引导图提示词

保存文件：YYYYMMDD_文章标题关键词_image_prompts.md
保存路径：当前工作目录的 `outputs/`，或用户明确指定的目录
```

同时把每张图的提示词作为引用块插入文章建议位置：

```markdown
> **插图提示词｜封面图：XXX**  
> Create a 900x383 horizontal WeChat article image...
```

尺寸规则：
- 封面图/分享预览图使用 `900x383`；
- 正文配图默认使用 `900x600`，不要做成细长横幅；
- 结尾引导图默认使用 `900x600`，除非明确要公众号底部横幅；
- 每个 JSON prompt 的 `size` 必须和用途一致。

Agnes 对中文文字渲染不稳定。默认图片 Prompt 不要要求模型生成中文、英文、数字、标签或任何可读文字；说明性文字放在正文、图注或 Markdown alt 文本里。只有用户明确要求“图片里必须有文字”时，才加入极少量文字，并提醒可能需要后期手动加字。

每张图的汇总文件里同时保留一个 JSON 代码块，便于 `scripts/generate_agnes_images.py --batch-json` 直接读取：

````markdown
## 01-cover

```json
{
  "filename": "02-inline.png",
  "size": "900x600",
  "prompt": "A 900x600 inline WeChat article illustration..."
}
```
````

**配图提示词模板：**
- 封面图：`image_templates/cover.md`
- 信息图：`image_templates/infographic.md`
- 引导图：`image_templates/cta.md`
- 风格基准：`image_templates/style-block.md`

**用户选择直接出图后才执行：**
- 使用 `scripts/generate_agnes_images.py` 调用 Agnes Image 2.1 Flash 逐张或批量生成。
- 不要把“直接画出来”解释为调用 GPT 图片模型或其他出图工具；除非用户明确要求临时 fallback，否则只使用 Agnes。
- API key 从环境变量 `AGNES_API_KEY` 或 skill 目录下 `config/agnes.env` 读取；不要把真实 key 写进文章或提示词文件。
- 不运行旧版 `scripts/generate_images.py`。
- 生成后把文章里的提示词引用块替换为 Markdown 图片链接。

---

## 注意事项

- 严格按照 `writing_style.md` 的风格写作
- **用完整句子表达，避免短句碎片化**
- 保持用户的语言风格
- 不要过度使用emoji（1-2个/篇即可）
- 专业术语要解释
- 数据建议标注来源
- 保持真诚，避免套路
- 目标字数：2500-3000字
- 重点标识使用行内引用反引号，不使用加粗
- **完成后自动保存到 `outputs/` 或用户指定目录**
